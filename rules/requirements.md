# 需求文档

> 记录用户需求、建议、参考信息
> 按时间顺序整理，标注优先级和状态

## Agent 更新义务

**当 agent 收到新需求或发现原需求有变更时，必须更新本文档。**
- 新需求提出时 → 立即添加记录
- 需求变更时 → 更新对应条目，标注变更时间和原因
- 需求完成时 → 更新状态为"已完成"
- 需求取消时 → 更新状态为"已取消"，记录原因

## 需求记录

### [YYYY-MM-DD] 需求标题
- **优先级**: P0/P1/P2
- **状态**: 待确认 / 进行中 / 已完成 / 已取消
- **描述**: 需求详情
- **备注**: 补充信息
