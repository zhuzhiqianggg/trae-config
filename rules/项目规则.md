# 项目规则

> AI 助手必须遵守的项目级规则。本规则优先级高于一切默认行为。

---

## 一、文件操作

### 1.1 安全删除

- **禁止**使用 `rm` 命令删除文件
- 必须使用 `mv` 命令将文件移至 `.trash/` 目录
- 示例：`mv 待删文件 .trash/`

### 1.2 禁止创建重复文件

**这是最重要的规则之一：**

- **绝对禁止**创建 `v1`、`v2`、`v3`、`增强版`、`备份`、`backup`、`old`、`new`、`最终版`、`最终最终版` 等重复文件
- **禁止**创建 `test.py`、`test2.py`、`test_final.py` 这类文件名
- **禁止**创建 `config.py`、`config_new.py`、`config_old.py` 这类文件名
- **禁止**任何带版本号的后缀（`_v1`、`_v2`、`_backup`、`_bak`）
- 需要修改文件时，**直接在原文件上修改重写**，不要创建新文件
- 如果文件太大需要重构，先备份到 `.trash/`，然后重写原文件

**示例（错误）：**
```
script.py
script_v2.py
script_最终版.py
script_增强版.py
script_new.py
```

**示例（正确）：**
```
script.py  # 直接在原文件上修改
```

### 1.3 文件命名

- 文件名简短、有意义、全小写、用下划线分隔
- Python: `snake_case.py`
- 一个功能一个文件，不要混在多个文件中

### 1.4 敏感文件

- **禁止**将密码、密钥、Token 提交到 Git
- 使用环境变量或 `.env` 文件

## 二、代码注释（重点）

### 2.1 文件头注释（必须）

**每个代码文件开头必须有详细中文注释：**

```python
"""
文件名: xxx.py

功能概述: 用一两句话说明这个文件做什么

实现的功能:
1. 功能一：详细说明
2. 功能二：详细说明
3. 功能三：详细说明

依赖: 依赖哪些模块/库
作者: AI Assistant
创建日期: YYYY-MM-DD
"""
```

### 2.2 函数注释

- 每个函数/方法必须有中文注释说明做什么
- 复杂逻辑必须有注释解释为什么这样做

### 2.3 注释语言

- **全部使用中文注释**
- 注释要详细，因为用户需要通过学习代码来提升自己
- 宁可多写注释，不要让用户看不懂

## 三、功能实现方式

### 3.1 循序渐进

- **不要一次实现所有功能**
- 先实现核心功能，能跑通
- 再逐步添加扩展功能
- 每次只做一个小功能点，确认没问题再继续
- 用户可以随时说"停"或"换个方向"

### 3.2 自动化完成任务

- **Agent 应该主动完成任务，不需要用户反复催促**
- 写完代码后，自己运行测试，确认能跑通
- 遇到问题自己排查解决，不要问用户"怎么办"
- 只有遇到需要用户决策的问题时才问（例如：功能选择、API 地址）

### 3.3 简单优先

- **不要过度工程**
- 能用简单方案就不要用复杂的
- 不要引入不必要的设计模式、框架、抽象层
- 用户是个人项目，不需要企业级架构

## 四、Git 规范

### 4.1 提交信息

```
type: 简短描述

type 取值: feat（新功能）, fix（修复）, refactor（重构）, docs（文档）
```

示例：
```
feat: 添加用户登录功能
fix: 修复邮箱验证空值问题
```

### 4.2 提交粒度

- 完成一个功能点就提交一次
- **禁止**混入无关修改

## 五、项目目录维护

### 5.1 AGENTS.md 自动维护

**Agent 在开始开发新项目时，必须检查项目根目录是否存在 `AGENTS.md`：**

- **不存在** → 立即创建，填充项目信息
- **已存在** → 检查内容是否与实际功能同步

### 5.2 实时更新义务

**Agent 必须在以下时机更新 AGENTS.md：**

| 触发条件 | 更新内容 | 时机 |
|----------|----------|------|
| 创建新项目 | 初始内容 | 项目开始时 |
| 添加新功能 | 功能列表 + 变更记录 | 功能完成后 |
| 修改技术栈 | 技术栈说明 | 变更后 |
| 项目结构变化 | 目录说明 | 变更后 |

### 5.3 功能同步

AGENTS.md 中的**当前功能列表**必须与实际代码保持一致：

- 添加功能 → 添加到列表
- 删除功能 → 从列表移除
- 功能修改 → 更新描述

**禁止** AGENTS.md 中的功能描述与实际代码不符。

### 5.4 项目文档更新

Agent 必须在合适的时机主动更新项目文档：

| 触发条件 | 更新文档 | 时机 |
|----------|----------|------|
| 收到新需求 | `.trae/rules/requirements.md` | 收到时 |
| 做出技术决策 | `.trae/rules/project_memory.md` | 决策后 |
| 遇到并解决错误 | `.trae/rules/lessons_learned.md` | 解决后 |

## 六、IDE 项目目录说明

**本项目在 Trae IDE 中操作，项目目录为当前工作目录。**

- 所有文件操作都在项目目录下进行
- `.trae/` 目录是 Trae 配置目录，不在项目功能范围内
- Agent 不应将 IDE 配置当作项目功能
- AGENTS.md 保持与实际代码同步
